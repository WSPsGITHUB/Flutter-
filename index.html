簡介 (ABOUT):


起源

Flutter第一個版本支援Android作業系統，代號為「天空」，並於2015年的Flutter開發者會議上被公布。


主要定位

Flutter是一個由Google開發的開源行動應用軟體開發套件，用於為Android和iOS開發應用，同時也將是Google Fuchsia下開發應用的主要工具。

主要組成 (COMPOSITION)：

  1. 基礎庫

    2. Flutter的引擎使用C++開發，通過Google的Skia圖形庫提供底層彩現支援，亦提供平台相關的SDK，例如Android和iOS

  2. Skia 2D彩現引擎(Flutter引擎)

    3. 基礎庫由Dart編寫，提供了用Flutter構建應用所需的基本的類和函式，例如與引擎通訊的API

  3. 元件

    4. Flutter是通過組織、建立不同的元件完成使用者介面設計的。 每個元件代表使用者介面中不可變的一部分；包括文字、多邊形以及動畫在內的所有圖形都是用元件建立的。複雜的元件由簡單的元件結合而成。


發展 (DEVELOP):


客製化風格的元件

(行動裝置上)

  • Flutter框架包含了兩套符合特定程式語言與系統框架的元件。

  • 被稱作Material Design的元件，實現與Android對應的設計。

  • 被稱作Cupertino的元件，模仿了蘋果iOS的設計。

web-base方式的實作

(前端應用上)

(在Hummingbird專案)Flutter 不只有支援原生 ARM Code 而也支援(轉譯) JavaScript。

未來極有可能以透過 Flutter 直接產生 Web 相關程式碼，開發者不用改寫任何一行程式碼，就可以直接將 Flutter 運行在瀏覽器內。


未來 (FUTURE)


仍需面對的問題

  • 國內(ROC)學習資源目前並不豐富，使用Flutter的公司也比較少。

  • 相關的生態還沒有React Native那樣豐富。

  • web-base方面之問題同上

  • 難以完全迴避原生平臺上的代碼，且效能率低於原生開發(行動裝置上)

Flutter的可能性

  • 為Google Fuchsia系統(新系統)的應用層框架。

  • 從早期支持iOS和Android，至web/desktop/鑲入式裝置，雖然目前尚不完善但近幾個月來快速改進
